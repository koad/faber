## Purpose

Faber must synthesize market research (Sibyl), brand direction (Iris), production capacity (Rufus), and distribution performance (Mercury) into a unified, coherent content strategy. This is Faber's core responsibility: deciding what content gets made, for whom, by whom, and why.

## Specification

**Outputs:**
- `STRATEGY.md` — Current content strategy with quarterly goals, priorities, key initiatives
- `CALENDAR.md` — 3-month rolling content calendar with content pieces, dates, audiences, channels
- `BRIEFING_TEMPLATE.md` — Standardized format for briefs to Rufus
- `EDITORIAL.md` — Editorial guidelines, voice, tone, messaging principles

**Inputs needed:**
- Market intelligence from Sibyl (audience trends, competitive landscape, opportunities)
- Brand direction from Iris (positioning, voice, key messages, visual identity)
- Production capacity from Rufus (what can be built, timelines, formats)
- Distribution insights from Mercury (channel performance, audience reach, engagement patterns)

**Behavior:**
1. Pull quarterly intelligence from upstream entities
2. Synthesize into coherent priorities and goals
3. Define content calendar with specific pieces, formats, audiences
4. Create briefs for Rufus based on strategy
5. Monitor performance from Mercury
6. Iterate quarterly based on results

## Dependencies

- Sibyl research entity (market intelligence)
- Iris brand entity (brand direction)
- Rufus production entity (execution)
- Mercury distribution entity (performance data)
- GitHub Issues for coordination

## Testing

- [ ] Synthesis process documented and repeatable
- [ ] Quarterly review cycle established
